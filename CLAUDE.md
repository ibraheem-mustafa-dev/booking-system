## Supabase Rules
- Row Level Security (RLS) REQUIRED on every table — no exceptions
- Write RLS policies before writing the table query
- Migrations only via `supabase/migrations/` directory — never edit tables directly in dashboard
- Foreign keys + indexes for all join columns
- Soft deletes (`deleted_at TIMESTAMPTZ`) — never hard delete user data

## TypeScript Standards
- No `any` types — use proper types or `unknown` with narrowing
- Zod validation on all API route inputs
- Server Components by default, Client Components only when needed (interactivity/hooks)
- `use server` actions for mutations — no client-side API routes

## Design Context

### Users
- **Primary:** Small business owners managing their own bookings (SGS clients, freelancers, clinics). Non-technical. They need to set things up fast and then forget about it.
- **Secondary:** Clients booking appointments. They arrive from LinkedIn, a website, or a direct link. They need to pick a date, fill a form, and leave — in under 60 seconds.
- **Context:** The business owner has ADHD. The interface must actively guide focus, not scatter it. Every screen should make the next action obvious without reading.

### Brand Personality
**Confident. High-energy. Unique.**
- Not corporate. Not playful. Not minimal-for-minimal's-sake.
- Think: a senior creative professional who knows their craft, moves fast, and doesn't waste time on unnecessary steps.
- The product should feel like it was made by someone opinionated, not generated by a template.

### Aesthetic Direction
- **Dashboard:** Dark, dense, premium. References: Linear, Vercel, Raycast. Orange-led accent palette.
- **Public booking pages:** Warm cream, textured, tactile. Like a well-printed invitation.
- **Anti-references:** Calendly (generic blue-grey), Cal.com (black-and-white minimal), Acuity (dated teal). Any shadcn/ui default theme. Any "warm off-white + serif heading" AI redesign.
- **Theme:** Always dark dashboard, always light public pages. No toggle.

### Design Principles
1. **Spotlight focus** — The thing you're interacting with is the brightest thing on screen. Everything else dims. Designed for ADHD, benefits everyone.
2. **Energy through motion** — Spring animations, immediate feedback, elements that feel alive. Nothing static, nothing sluggish.
3. **Orange is the brand** — Almost nobody in the booking space uses warm orange. It's energetic, approachable, and instantly recognisable.
4. **Connected, not corridors** — Information should be linked and accessible from context. Future: command palette, slide-over panels. Present: actionable stats, inline actions.
5. **Precision in data** — Monospace numbers (JetBrains Mono) for all times, prices, stats. Signals "this is a real tool" without being cold.

## Design System — "Dark Confidence"

The booking system must feel premium, confident, and high-energy. Not generic SaaS. Not clinical. Not "warm corporate." It should be immediately distinguishable from Calendly, Cal.com, and Acuity.

### Visual Identity

**Two surfaces, two treatments:**
- **Dashboard (business owner):** Dark background, dense, powerful. Think Linear, Vercel, Raycast.
- **Public booking pages (client-facing):** Light but textured and warm. Like a well-printed invitation, not a government form.

**Colour palette — orange-led, not teal-led:**
- Primary CTA/highlight: `#F87A1F` (warm orange) — the differentiator. Almost no booking tools use orange.
- Secondary/info: `#0F7E80` (teal) — for informational elements, available slots, confirmation states.
- Dashboard background: `#0C0C0F` (near-black), cards: `#18181B` (dark zinc).
- Public page background: `#FAF8F5` (warm cream) with subtle CSS noise texture.
- Status colours: semantic (green/red/amber) but warm-tinted.

**Typography:**
- Headings: Inter 700, tight tracking (-0.02em). Confident, not decorative.
- Data/numbers (stats, times, prices, dates): JetBrains Mono. Monospace = precision.
- Body: Inter (default). No serif headings — that's the AI redesign starter pack.
- Both fonts loaded via `next/font/google`.

**Layout principles:**
- Bento grid for dashboard stats (varied card sizes, not uniform). One hero stat, mixed medium/tall.
- Recent bookings as individual row cards with coloured status stripe, not HTML tables.
- Per-booking-type theming via `--brand-*` CSS variables (already partially implemented).

### ADHD-Friendly UX Principles

These are non-negotiable for all UI work on this project:

1. **Focus isolation:** The element you're interacting with brightens/lifts. Everything around it dims to ~60% opacity. Not subtle — noticeable. Like a spotlight.
2. **Clear separation:** Every interactive region must have visible boundaries. No ambiguous whitespace between clickable areas.
3. **High-contrast states:** Hover, active, and focus states must be visually obvious — brightness shifts, scale changes, colour changes. Never just a border-colour tweak.
4. **Non-linear navigation (Phase 2):** Command palette (`Cmd+K`) for jumping anywhere. Contextual slide-over panels instead of page navigation for detail views. The UI should feel connected, not like walking down corridors.
5. **Immediate feedback:** Every interaction gets instant visual response. Spring-based animations (~200ms, slight overshoot), not linear transitions.

### Interaction States

All interactive elements must implement these states:
- **Hover:** Element lifts (translateY -2px), orange glow border, siblings dim
- **Focus:** Stronger glow, visible orange focus ring (WCAG 2.2 AA compliant)
- **Active/pressed:** Scale down to 0.97, instant snap
- **Selected:** Solid orange left border + brighter background
- **Loading:** Pulsing orange dot sequence (three dots), not a spinning circle
- **Transitions:** Spring-based (CSS `cubic-bezier(0.34, 1.56, 0.64, 1)`), never `ease-in-out`

### Public Booking Flow Aesthetic

- Page wrapper: cream background with CSS noise texture
- Calendar: available dates get an orange dot indicator beneath them
- Time slots: pill-shaped, flex-wrap layout (not grid), hover slides right with orange border
- Form inputs: bottom-border only, floating labels on focus
- Confirmation: teal circle + white tick with pop animation, copy says "You're booked in" not "Booking confirmed"

### What NOT to Do

- No serif heading fonts (DM Serif Display, Playfair, etc.) — AI redesign cliché
- No warm off-white + teal as the primary palette — that's every "make it less boring" prompt
- No thin grey borders on white cards — that's the Calendly/shadcn default
- No spinning circles for loading states
- No linear `ease-in-out` transitions
- No uniform card grids — use varied sizes

## Product Vision (Future Phases)

For context when making design decisions — this is where the product is heading:

- **Phase A (current):** Visual redesign to "Dark Confidence" aesthetic
- **Phase B:** Onboarding system — setup wizards, guided tutorials, contextual tooltips
- **Phase C:** AI chatbot — guidance, support, booking automation. N8N integration with knowledge docs + RAG for support. Natural language recipe matching.
- **Phase D:** Freemium model — core features free, extras (chatbot automation, advanced analytics) as paid upgrades
- **Phase E:** SGS WordPress booking block — iframe embed widget for client sites

The system should work for any booking type (consultancy, music lessons, medical appointments) while feeling specialised to each through per-booking-type theming.
